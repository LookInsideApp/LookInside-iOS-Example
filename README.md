# LookInside-Example

> Public SwiftUI demo fixture for trying LookInside end to end on iOS and macOS.

A SwiftUI app used as the end-to-end test target for the LookInside macOS host and the prebuilt server runtime published by [`LookInside-Release`](https://github.com/LookInsideApp/LookInside-Release). Three richer screens cover the layouts most people see every day: a music player, a social feed, and a chat client.

The checked-in Xcode project is generated by Tuist and intentionally kept in git so users can build the app without installing Tuist. By default it consumes the public `LookInsideServerStatic` SwiftPM binary product from `LookInside-Release`.

## Build

iOS Simulator:

```bash
xcodebuild -scheme LookInsideExample \
           -project LookInsideExample.xcodeproj \
           -destination 'platform=iOS Simulator,name=iPhone 16' \
           -skipMacroValidation -allowProvisioningUpdates build
```

macOS:

```bash
xcodebuild -scheme LookInsideExample \
           -project LookInsideExample.xcodeproj \
           -destination 'platform=macOS' \
           -skipMacroValidation -allowProvisioningUpdates build
```

Or open `LookInsideExample.xcodeproj` in Xcode and pick the destination from the run scheme.

The Makefile wraps these:

```bash
make build-sim    # iOS Simulator (generic)
make build-mac    # macOS
make run          # iOS Simulator: build + install + launch
make run-mac      # macOS: build + open the .app
```

## Code signing

Tuist wires Xcode build settings through `Configuration/*.xcconfig`. The checked-in defaults use automatic signing with an empty development team, so simulator and local Mac builds work without a signing setup.

For physical-device runs (or notarization), create a local `Configuration/custom.xcconfig`. Debug and Release both include this file automatically when it exists. It is gitignored and is the place to put machine- or team-specific signing overrides:

```xcconfig
DEVELOPMENT_TEAM = YOURTEAMID
```

## Regenerate the Xcode project

Tuist is only needed when changing the project structure.

```bash
tuist generate --no-open
```

The default generated project uses the remote binary dependency from `LookInside-Release`. To generate against a local source checkout of a compatible server package, set `TUIST_LOOKINSIDE_SERVER_PATH`:

```bash
TUIST_LOOKINSIDE_SERVER_PATH=/path/to/LookInsideServer tuist generate --no-open
```

When using the Makefile wrapper, pass `LOOKINSIDE_SERVER_PATH=/path/to/LookInsideServer make generate`.

The local package must expose the `LookinServer` library product. The generated `.xcodeproj` should remain checked in after regeneration.

## E2E flow

1. Build and run this app on the iOS Simulator (or as a Mac app).
2. On the same Mac, run the `LookInside` macOS host app.
3. The host auto-discovers the running app via Peertalk on `47164–47169` (sim) or `47175–47179` (USB).
4. Click any view in the host to inspect the live UIKit/SwiftUI hierarchy.
5. SwiftUI hierarchy inspection requires an activated host.
